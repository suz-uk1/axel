AI TRACE Checklist

Use case → Risk → Control → Evidence. If you can’t trace your top AI risks to real controls and timestamped evidence, procurement will treat “responsible AI” as marketing.

Download (CSV) ISO27001 TRACE

No signup. This is a lightweight governance operating checklist (ISO 42001-ready).

What this prevents

  • “We have a policy” with no evidence of practice
  • Unknown AI inventory (shadow GenAI usage)
  • RAG / training data risk blind spots (PII/IP leakage)
  • No monitoring/incident loop for harmful outputs

How to use (15 minutes)

  • Pick your top 5 AI use cases (customer-facing first)
  • List top risks per use case
  • Write 1–2 controls per risk and link evidence
  • Schedule a cadence (monthly/quarterly) and log runs

TRACE row (minimum columns)

Use case → Risk → Control → Owner → Cadence → Evidence link → Last run date

Evidence should be reproducible (ticket/log/report/runbook), not screenshots.
