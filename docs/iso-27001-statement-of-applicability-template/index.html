ISO 27001 Statement of Applicability (SoA) Template (ISO 27001:2022)

The Statement of Applicability (SoA) is where ISO 27001 becomes auditable: it lists which Annex A controls apply, why, and what evidence proves implementation.

Tip: auditors want traceability: scope → risks → SoA → procedures → evidence.

What the SoA is (and why auditors care)

  • Lists Annex A controls (ISO/IEC 27001:2022 Annex A)
  • Marks each control as applicable / not applicable
  • Documents justification (risk/context/requirements)
  • Tracks implementation status
  • Points to objective evidence (policies, logs, tickets, configs)

Startup-friendly SoA columns

  1. Control ID
  2. Control name
  3. Applicable? (Y/N)
  4. Justification (1–3 sentences)
  5. Implementation status
  6. Owner
  7. Evidence / links
  8. Notes (optional)

How to complete the SoA (fast)

  1. Lock scope first
  2. Run risk assessment
  3. Select controls based on risk treatment
  4. Write applicability + justification in plain language
  5. Add evidence pointers (what you’ll show at Stage 2)
  6. Review for contradictions (scope vs assets vs risks vs SoA)
