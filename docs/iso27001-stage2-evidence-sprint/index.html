ISO 27001 Stage 2 Evidence Sprint (7 Days)

Stage 2 is an evidence game. Policies help, but auditors certify proof. If you’re a small team, the fastest path is to run evidence like a sprint.

If you want the ready-to-fill assets too, here’s the pack: ISO 27001 Stage 2 Evidence Pack (preview).

How to use this

  1. Pick a single owner for the sprint.
  2. Create one folder (or drive) where evidence lives.
  3. For each control area, capture: what happened, who approved, and where the record is.

7-day plan (minimal, startup-friendly)

Day 1 — Evidence map + scope

  • Define scope (systems, repos, vendors).
  • Create an evidence map (control → evidence → owner → link).

Day 2 — Access control proof

  • User access review record (who reviewed, when, result).
  • Joiner/mover/leaver process evidence (ticket examples).

Day 3 — Asset + change management proof

  • Asset inventory export + ownership.
  • Change approvals (PR reviews, release logs, approvals).

Day 4 — Incident management proof

  • Incident response procedure + one tabletop/drill record.
  • Postmortem template + at least one example (sanitized is fine).

Day 5 — Supplier security proof

  • Vendor list + risk tiering.
  • Due diligence evidence for top vendors (SOC2/ISO certs, DPAs).

Day 6 — Training + awareness proof

  • Security training record (who completed, when).
  • Phishing/awareness comms (or policy acknowledgement record).

Day 7 — Internal audit + management review proof

  • Internal audit checklist + findings log.
  • Management review minutes (even a lightweight one-page record).

Quick checklist (what auditors typically ask to see)

  • Access review evidence (date, reviewer, result)
  • Change approvals / release history
  • Incident drill or incident record
  • Supplier due diligence records
  • Training completion evidence
  • Internal audit + management review evidence
