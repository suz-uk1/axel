Minimal fields (per vendor)

  • Vendor name
  • Service / purpose
  • Data types (PII? credentials? customer content?)
  • Access level (prod access? admin access?)
  • Criticality (Low/Med/High)
  • Security assurances (SOC 2/ISO 27001/pen test, etc.)
  • Risk decision (Accept / Mitigate / Replace)
  • Mitigations / requirements (e.g., SSO/MFA, contract clauses)
  • Owner + review date
  • Evidence link(s)

Auditor tips

  • Focus on your critical vendors first (hosting, payments, auth, support tools).
  • One completed review with evidence beats a long questionnaire nobody maintains.
  • Link to DPAs/contracts and any assurance reports you have access to.
