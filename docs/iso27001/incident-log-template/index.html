Minimal incident log fields

  • ID
  • Date/time detected
  • Category (phishing / access / malware / outage / data issue)
  • Severity (Low/Med/High)
  • Description (1–2 lines)
  • Impact summary
  • Actions taken
  • Owner
  • Closure date
  • Post-incident improvement / CAPA link
  • Evidence link (ticket, Slack thread, screenshots, etc.)

If you had “no incidents”

  • Keep the log anyway.
  • Add a periodic review entry (e.g., monthly) with “no incidents reported”.
  • Link the incident procedure/runbook as evidence.
