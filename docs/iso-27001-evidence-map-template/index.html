How to use

  • Start with your key Annex A controls (and any clause requirements you want to evidence).
  • For each control, write a 1–2 sentence “what we do”.
  • Link the evidence (ticket, log, runbook, screenshot folder, vendor doc).
  • Add cadence + owner so evidence exists before the auditor asks.

What auditors like

  • Specific evidence locations (not “we do this”)
  • Repeatable cadence (monthly/quarterly)
  • Traceability to risks/SoA decisions

CSV columns

This template contains the minimum fields for a startup-sized evidence map.

  • control (Annex A / clause)
  • applicable (Y/N)
  • what_we_do
  • evidence_location (URL/folder/tool path)
  • owner
  • cadence
  • last_checked
  • gaps_notes
