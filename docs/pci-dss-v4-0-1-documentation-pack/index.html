The problem

  • Teams waste weeks writing “generic” policies that don’t match reality.
  • Procedures are undocumented (until the week before the audit).
  • Evidence isn’t collected continuously → panic at audit time.

The solution

  • Minimal-but-defensible documentation baseline for startups/SMBs.
  • Tailoring prompts so each doc fits your environment.
  • Evidence-first checklists (what to save, where it usually lives).

What’s inside (v0.1)

  • Core policy pack (12–18 docs): IS policy, access control, logging/monitoring, change mgmt, secure development, incident response, vendor mgmt, retention, crypto/key mgmt, backup/BCP, acceptable use, risk method…
  • Operational procedures pack (10–15 docs): access reviews, user provisioning, vuln mgmt cadence, patching, log review, incident steps, evidence capture, exceptions…
  • Evidence pack: evidence request list + “minimum viable evidence” for first readiness pass
  • 30-day readiness plan: docs → adoption → evidence → gap review
