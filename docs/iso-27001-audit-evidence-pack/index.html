ISO 27001 Audit Evidence Pack (Stage 2) — Checklist

Stage 2 audits get painful when evidence is scattered. The easiest way to pass is a simple evidence map: control → evidence → where it lives.

What “good evidence” looks like

  • Objective (logs, tickets, configs, screenshots, reports)
  • Traceable (SoA → procedure → evidence)
  • Time-bounded (covers the audit sample period)
  • Reviewable (auditor can follow the link and understand context)

Evidence map template (recommended columns)

  1. Control / topic
  2. Evidence item
  3. System/source (Jira, IAM, MDM, Git, SIEM, etc.)
  4. Owner
  5. Sample period
  6. Link
  7. Notes

Fast wins auditors frequently sample

  • Access control: joiner/mover/leaver evidence + periodic review
  • Change management: ticket/PR → approval → deploy evidence
  • Backups: restore test evidence
  • Incidents: 1 incident end-to-end + CAPA
  • Suppliers: 1 critical vendor end-to-end
  • Internal audit + management review minutes
