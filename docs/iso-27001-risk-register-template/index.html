Recommended columns

  1. Risk ID
  2. Asset / process
  3. Threat
  4. Vulnerability
  5. Impact (CIA)
  6. Likelihood
  7. Inherent risk score
  8. Existing controls
  9. Risk treatment decision
  10. Treatment plan (actions)
  11. Target date
  12. Owner
  13. Residual risk score
  14. Evidence / links

How to fill it (fast, audit-consistent)

  1. Start with scope (systems/teams in ISMS)
  2. List key assets/processes (not everything)
  3. Identify top threats that match your reality
  4. Score impact/likelihood consistently
  5. Decide treatment + assign owners/dates
  6. Link treated risks to SoA controls + evidence
