EU procurement security questions (and what evidence to show)

Many startups don’t fail ISO 27001 because of “missing policies.” They get stuck because EU customers send a vendor questionnaire and the team can’t quickly produce a clean evidence trail. This page is a practical list of the questions that tend to come up — and the minimum evidence set that answers them.

Common questionnaire sections

1) Data protection (GDPR)

  • Do you provide a DPA?
  • Do you use subprocessors? Do you publish a subprocessor list?
  • Do you offer EU data residency?
  • Do you sign SCCs when needed?

Evidence: DPA template, subprocessor list, hosting architecture note, SCC process record.

2) Access control

  • Who has production access? How is access granted/revoked?
  • Do you do periodic access reviews? Show the last one.
  • MFA/SSO enforcement and privileged access controls?

Evidence: access review log + screenshots/exports + onboarding/offboarding record.

3) Vulnerability management

  • Scanning cadence and SLA?
  • Show an example finding → fix → verification.

Evidence: vuln log + remediation tickets + patch report.

4) Incident response

  • Do you run tabletop exercises?
  • What’s your breach notification process?

Evidence: IR plan + incident drill record + postmortem template.

5) Change management

  • How are changes approved? Is there code review?
  • Who can change security configurations (logging, IAM, infra)?

Evidence: PR review examples + change approval log + release notes.
