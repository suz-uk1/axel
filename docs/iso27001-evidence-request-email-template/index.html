Copy-paste email

Subject: Security evidence request (procurement / ISO 27001)

Hi {{Name/Team}},
We are responding to a customer procurement / security review and need the following evidence artifacts. Links are preferred (docs/portals). Please include last updated dates where available.

Requested artifacts

  1. Security policy set (InfoSec Policy + access control + incident response)
  2. Access controls: onboarding/offboarding process + latest access review record
  3. Change management: PR approval process + release/change log
  4. Incident response: IR plan + a recent drill/tabletop or postmortem (sanitized)
  5. Vulnerability management: scanning cadence + remediation SLAs + recent summary
  6. Business continuity: backups + restore test evidence
  7. Third-party / supplier security: vendor list + due diligence criteria (SOC2/ISO, DPA, reviews)
  8. Audit & review: internal audit checklist + management review minutes (or summary)

If you have a trust center or preferred portal, please share the link and access instructions. Otherwise, a shared folder with the above artifacts is perfect.

Thanks,
{{Your name}}

Pro tip: send links, not PDFs

  • Provide one folder link and keep filenames consistent.
  • Add “Last updated” dates (buyers care about freshness).
  • Keep an Evidence Map so every request maps to a row.
