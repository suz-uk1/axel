ISO 27001 Management Review

What auditors want to see

  • It happened (date, attendees, scope)
  • Inputs were reviewed (KPIs, incidents, audit results, risks)
  • Decisions were made (actions, owners, dates)
  • Outputs are tracked (follow-up next review)

Startup-friendly cadence

  • Quarterly is enough for most early teams
  • Do a 30–45 min structured meeting
  • Attach links to evidence instead of long docs

Simple agenda (copy/paste)

  1. Scope changes (products, systems, vendors)
  2. Risk register changes + risk acceptance decisions
  3. Incidents + near-misses + lessons learned
  4. Internal audit results + CAPA status
  5. Control performance (access reviews, backups/tests, training)
  6. Supplier/security changes
  7. Objectives/KPIs (what improved, what regressed)
  8. Decisions + actions (owner + due date)

Common mistakes

  • Minutes without decisions → add actions with owners/dates.
  • No linkage to risks → at least review top 5 risks each time.
  • All talk, no evidence → attach links/screenshots/log extracts.
