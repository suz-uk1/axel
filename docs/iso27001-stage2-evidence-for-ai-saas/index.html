Top auditor questions (AI SaaS edition)

  • Access: Who has prod access, how is it reviewed, and where’s the last review record?
  • Change control: Show a change that touched security controls and its approval trail.
  • Vuln mgmt: What’s your SLA? Show an example vuln → fix → verification.
  • Logging: What do you log, who can change logging config, and how do you respond to alerts?
  • Incidents: Do you run tabletop exercises? Show one test and lessons learned.
  • Suppliers: How do you evaluate cloud/LLM vendors? Show a due-diligence record.
  • AI governance: How do you prevent prompt/data leakage and control model/tool access?

Minimal evidence set (copy/paste structure)

If you only build one spreadsheet, build an evidence map with these columns:

  • Control / Annex A ref
  • Evidence item (log/report/screenshot/doc)
  • Owner
  • Frequency
  • Link (Drive/Notion/Jira/GitHub/etc)

Example + filled row set is included in the Evidence Pack.

7-day plan (stage 2 sprint)

  1. Day 1: scope + SoA draft + evidence map skeleton
  2. Day 2: access review + asset inventory + supplier list
  3. Day 3: vuln mgmt evidence + patch SLAs + sample remediation
  4. Day 4: logging/monitoring evidence + alert runbook
  5. Day 5: change mgmt evidence + secure SDLC artifacts
  6. Day 6: incident tabletop + backup/restore evidence
  7. Day 7: internal audit checklist + management review notes

If you want the exact artifacts to produce each day, use the pack’s sprint doc.
