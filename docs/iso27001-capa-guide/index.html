What CAPA is

CAPA = Corrective and Preventive Action. For ISO 27001, treat it as a simple system for:

  • Logging nonconformities (audit findings, incidents, process failures)
  • Finding root cause (not symptoms)
  • Implementing fixes + verifying effectiveness

Minimum fields (startup-sized)

  • ID + date
  • Issue description (what happened)
  • Source (internal audit / incident / customer / etc.)
  • Root cause
  • Corrective action (what you changed)
  • Owner + due date
  • Status
  • Effectiveness check (how you know it worked)
  • Evidence link

Simple workflow (no bureaucracy)

  1. Log the issue (1–2 sentences).
  2. Contain immediate risk (quick fix if needed).
  3. Root cause: why did this happen in the system?
  4. Fix: change the process/control (not just the document).
  5. Verify: confirm via evidence (test, log, re-audit).
  6. Close only when evidence exists.

Common mistakes

  • "Fixed" means updated a policy → auditors look for operational evidence.
  • No root cause → same issues recur, findings repeat.
  • No effectiveness check → can’t prove prevention.
