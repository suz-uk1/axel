SOC 2 → ISO 27001 mapping (startup-friendly)

If you already have SOC 2 (or you’re close), you can reuse a lot — but ISO 27001 Stage 2 still expects an evidence trail (not just policy PDFs). This page maps what transfers, what doesn’t, and the shortest path to a clean Stage 2 evidence set.

Fast path (what to do this week)

  1. Build an evidence map (controls → evidence items → owners → links).
  2. Lock the SoA: scope, applicability, and justification.
  3. Collect Stage 2 evidence: access reviews, vuln mgmt, change approvals, incident tests, supplier due diligence, etc.

What SOC 2 usually covers well (and transfers to ISO 27001)

  • Access control basics: SSO/MFA, onboarding/offboarding, least privilege.
  • Logging/monitoring: centralized logs, alerting, incident response workflow.
  • Vulnerability management: scanning cadence, patch SLAs, triage process.
  • Change management: approvals, PR reviews, deployments, rollback.
  • Vendor / subprocessor review (to a point): questionnaire, contracts, DPIA-ish artifacts.

If your SOC 2 evidence is clean, you can reuse artifacts — but ISO auditors often want a clearer control→evidence trace and the ISO-specific structure (SoA, risk treatment, internal audit cadence).

What often does not transfer cleanly

  • Statement of Applicability (SoA): ISO 27001’s required format + justification discipline.
  • Risk assessment / treatment plan: ISO expects a formalized cycle and linkage to controls.
  • Internal audit + management review: evidence that the ISMS is operating (not just designed).
  • Scope + asset inventory: auditors may go deeper than SOC 2 scope docs.

Minimal mapping table (practical, not academic)

Use this as a starting point for your evidence map.

  • SOC2 CC6.* → ISO 27001 Annex A (access control / identity) + Stage 2 evidence: access reviews
  • SOC2 CC7.* → ISO (monitoring / incident mgmt) + evidence: incident drill, alerts, postmortems
  • SOC2 CC8.* → ISO (change mgmt) + evidence: approvals, release notes, rollback tests

If you want a ready-made structure: use the Stage 2 Evidence Checklist and then fill it with your links.

When you should skip ISO 27001 (even if you have SOC 2)

  • You only need SOC 2 for customer procurement this quarter.
  • Your scope is unstable (infra/platform changes weekly).
  • You can’t support an internal audit cadence yet.

In that case, ship SOC 2 first — but keep your evidence map format compatible with ISO later.
