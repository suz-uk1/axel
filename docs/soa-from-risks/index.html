Build your SoA from real risks (worksheet)

Most teams create a generic Statement of Applicability (SoA) first, then try to justify it later. Auditors and procurement can smell this immediately. This worksheet forces a clean trace: Risk → Control (SoA claim) → Owner/Cadence → Evidence.

How to use (10 minutes)

  1. Write your top 5 risks tied to in-scope assets.
  2. For each risk, pick the minimum viable controls you can defend (Annex A IDs).
  3. Set owner + cadence for each control, and link one evidence artifact.
  4. In internal audit, sample these 5 rows first. If they trace cleanly, the rest is easier.

Worksheet columns

risk_id, risk_one_liner, asset/threat/impact,
control_ids (Annex A), soa_applicability (Y/N), soa_justification,
control_owner, cadence, evidence_link, last_run_date, notes

Tip: keep SoA justification short and tie it to risk treatment choices.
