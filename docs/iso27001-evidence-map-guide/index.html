ISO 27001 Evidence Map

ISO 27001 is an evidence game. An evidence map prevents the “we do it, but can’t prove it” scramble right before Stage 2.

Go to ISO 27001 hub Get the full Toolkit

What an evidence map is

An evidence map is a table that links:

  • Control (Annex A or clause requirement)
  • Implementation (what you do in practice)
  • Evidence (where proof lives)
  • Cadence (how often it’s produced/checked)
  • Owner (who maintains it)

What auditors ask for

  • Access control: joiner/mover/leaver records, access reviews
  • Change management: tickets/PRs, approvals, logs
  • Incidents: incident reports, postmortems, corrective actions
  • Suppliers: vendor review notes, contracts, security checks
  • Backup/DR: backup jobs, restore test evidence
  • Internal audit + management review minutes

Startup-friendly structure (copy/paste columns)

  • Control
  • Applicability (Y/N)
  • What we do
  • Evidence link/location
  • Owner
  • Cadence
  • Last checked
  • Notes / gaps

Common mistakes

  • Evidence is “in someone’s head” → write the location (folder, tool, link).
  • No cadence → add a schedule (monthly/quarterly) so evidence exists when asked.
  • Too broad → keep it minimal; add detail only where auditors probe.
