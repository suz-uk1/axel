Top 5 auditor questions (Stage 2)

  • Who owns each control? (Named owner, not “the team”.)
  • Show me evidence — can you open the link/screenshot right now?
  • How do you prove it’s current? (Last updated + retention + timestamps.)
  • How do changes get approved? (PR rules / tickets / approvals.)
  • What happens when things go wrong? (Incident tickets + postmortems + corrective actions.)

This pack is built to answer these quickly: owners + links + timestamps + a 7‑day plan.

7‑day evidence sprint (preview)

A tiny preview of the plan inside the pack:

  1. Day 1: pick the top 20–30 controls in scope + assign evidence owners.
  2. Day 2: access & identity evidence (MFA, joiner/leaver, access review).
  3. Day 3: SDLC + change management (PR rules, CI/CD logs, vuln workflow).
  4. Day 4: logging/monitoring (what’s logged, retention, alerts, incident tickets).
  5. Day 5: backup & recovery (config + restore test evidence).
  6. Day 6: suppliers + assets (vendor list, security reviews, inventory).
  7. Day 7: internal audit rehearsal (open 10 random links; fix broken permissions).
