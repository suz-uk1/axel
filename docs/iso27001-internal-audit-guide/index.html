ISO 27001 Internal Audit

Internal audit is not a “box to tick.” It’s your proof that the ISMS is operating, not just documented.

What auditors expect

  • An internal audit program (plan/scope/cadence)
  • Competence + independence (as reasonable for your org size)
  • Audit evidence (checklists, notes, sampling, records reviewed)
  • Findings (including “no findings” is acceptable)
  • Corrective actions (CAPA) for nonconformities

Startup-friendly cadence

You don’t need a massive audit department. You need consistency.

  • Monthly: mini-audit (1–2 processes, 30–60 min)
  • Quarterly: audit a control cluster end-to-end (e.g., access + joiner/mover/leaver)
  • Pre-cert: full sweep of clause 4–10 + key Annex A controls

Minimum artifacts (what to save)

  • Audit plan (what/when/who)
  • Checklist (what you checked)
  • Evidence list (links to tickets/logs/screenshots/policies)
  • Audit report (summary + findings + actions)
  • CAPA entries for any findings

Common mistakes

  • Audit only documents → auditors want operational evidence.
  • No sampling → pick a few recent changes/incidents and trace them.
  • Findings never become actions → you need CAPA with owners/dates.
